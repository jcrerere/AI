# facilities

这里放的是 `设施/地点锚点` 条目，不是把所有地名都拆成独立世界书。

后续新增地点时，统一按下面的规则写：

- `区域条目` 负责挂本区的核心地点名。目的不是展开细节，而是让“提到地点时，也能顺带带出所属区域背景”。
- `设施条目` 只挂地点自身、功能词、机构词、服务词，不挂区域名。目的不是让“提到地区时触发地点”，而是只在真正谈到这个地点时才展开细节。
- `机制条目` 负责挂分区映射、编号设施、通行规则、税务规则这类抽象规则，不替代具体地点条目。

建议只把下面两类地点单列成设施条目：

- `核心锚点设施`
  例：真理部大楼、白湾会馆、罪吻广场、潮镜别墅。
  这些地点有稳定功能、明确机构归属、会反复出现在剧情里。
- `设施体系代表条`
  例：X号性控所、HX男奴公寓、跨区关卡桥梁网络。
  这些不是单栋建筑，而是一整套编号设施或网络，需要单独说明规则。

不要单列的情况：

- 普通街巷、临时店铺、一次性酒吧、随机旅馆
- 只是区域氛围补充，但不会长期复用的背景地点
- 只有景观意义、没有制度功能和剧情接口的地方

推荐触发策略：

- `谈地区`：只进区域条目，知道本区大概有哪些代表地点，不展开每个点的细节
- `谈地点`：同时进设施条目和区域条目，既拿到地点细节，也拿到法域/区域背景
