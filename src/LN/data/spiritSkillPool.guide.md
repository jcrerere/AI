# Spirit Skill Pool Guide

## 目标

LN 的灵弦系统分三层：

1. `前端灵弦池`
- 维护完整卡池
- 负责种族锁、性别锁、破锁、等级、卡面描述、AI 摘要
- 用于开局选择、前端展示、运行时补全

2. `角色世界书条目`
- 不背整个灵弦池
- 只声明该角色当前已挂载的灵弦、挂载部位、等级/传说/灵核标签
- 不直接展开灵弦完整效果，完整效果与等级差异以前端当前等级卡面为准

3. `AI 注入层`
- 不让 AI 只凭灵弦名字猜效果
- 前端会根据当前挂载，从灵弦池补全 `aiSummary / description / effectLines / 锁来源`
- 再把补全后的摘要注入隐藏 prompt

## 锁池分类

当前锁池分类统一为：

- `通用池`
- `女性通用`
- `男性通用`
- `人类女性特有`
- `人类男性特有`
- `种族专属`
- `XX破锁`

说明：

- `种族专属` 不要求单独建世界书条目
- 是否专属、是否可破锁，统一由前端卡池控制
- 世界书只写角色现在真正持有的灵弦

## 卡池字段模板

前端灵弦卡建议统一使用以下字段：

```ts
type SpiritSkillCard = {
  id: string;
  name: string;
  level: 1 | 2 | 3 | 4 | 5;
  category?: 'common' | 'species' | 'female' | 'male';
  raceLocks?: string[];
  genderLocks?: Array<'male' | 'female'>;
  breakUnlocks?: Array<{
    label: string;
    raceLocks?: string[];
    genderLocks?: Array<'male' | 'female'>;
    note?: string;
  }>;
  slotHints?: string[];
  description: string;
  effectLines: string[];
  aiSummary: string;
  sixDimBonuses?: Partial<Record<'力量' | '敏捷' | '体质' | '感知' | '意志' | '魅力', number>>;
};
```

## 编写原则

### 1. 灵弦名称是池子定义，不是角色用法

正确：

- `魅气`
- `灵孢寄生`
- `芬里尔之眼`
- `灵魂雕塑者`

不要把角色用法直接写进池子名称：

- `足底魅气`
- `失联灵孢寄生`
- `雨雨版灵魂雕塑者`

这些应写在角色条目里，作为挂载部位与角色特化用法。

补充：

- `灵魂雕塑者`属于灵核特例，不走 `Lv1~Lv5` 成长线，前端以单卡形式维护，显示标签为`灵核`
- `奴隶烙印`、`月蚀王权`、`万有引力`、`御水术`、`女神之赐`、`血肉圣化`、`死神`与`活体摘器`也属于灵核特例，不走 `Lv1~Lv5` 成长线，前端以单卡形式维护，显示标签为`灵核`
- `魅气统御`与`血狼猎息`也属于传说级特例，不走 `Lv1~Lv5` 成长线，前端以单卡形式维护，显示标签为`传说`
- `影祷降身`也属于传说级特例，不走 `Lv1~Lv5` 成长线，前端以单卡形式维护，显示标签为`传说`
- `血肉丰缩`保留为身躯传说词条，不归灵核
- 角色若存在“灵核能力”，可作为独立于普通挂载灵弦之外的灵核能力记录；角色条目只写名字与灵核标签
- `呼吸剥夺`不再作为独立灵弦维护，而并入`夺息 Lv5` 的满载阶段

### 2. 部位不进技能名，部位进挂载表

正确示例：

```yaml
双足:
  - 名称: 魅气
    等级: Lv5
```

不要写成：

```yaml
双足:
  - 名称: 足底魅气
```

### 3. 等级是同技能不同强度，不要拆成不同技能

例如：

- `魅气 Lv1`
- `魅气 Lv5`

是同一条灵弦的不同等级，不应写成不同家族技能。

### 4. 破锁写法

破锁用于表达：

- 原本不属于当前种族/性别池
- 但存在特定例外路径

示例：

```ts
breakUnlocks: [
  {
    label: '人类女性破锁',
    raceLocks: ['人类'],
    genderLocks: ['female'],
    note: '科研与频段注入导致的低概率觉醒。',
  },
]
```

## 角色条目写法

角色条目内只写当前持有挂载：

```yaml
已持有灵弦:
  双眼:
    - 芬里尔之眼 Lv4
  双足:
    - 魅气 Lv5

灵核能力:
  - 灵孢寄生 Lv4
  - 灵魂雕塑者 灵核
```

原则：

- 写 `部位`
- 写 `等级/传说/灵核`
- 不重复整个卡池定义
- 不把来源、用法和更高等级的完整能力偷写进角色条目

## AI 为什么不会只靠名字乱猜

请求 AI 前，前端会：

1. 读取当前角色已挂载灵弦
2. 到灵弦池里按名字和部位匹配卡片
3. 用池子里的：
- `aiSummary`
- `description`
- `effectLines`
- `来源标签`
补全挂载信息
4. 再把补全后的摘要注入隐藏 prompt

因此：

- 世界书里只写名字，不够
- 世界书 + 前端卡池 + 运行时挂载，三者一起才稳

## 剧情关系线是否必须

不是。

建议分两层：

### 核心角色

适合补剧情关系线：

- 会反复出场
- 会影响主线
- 有长期关系演化
- 有灵网 / 离屏补帧 / 特殊回收逻辑

例如：

- 雨雨
- 康纳兰
- 十三使徒核心成员

### 开放世界普通角色

不必强绑关系线：

- 只有基础设定
- 只要有种族、身份、灵弦、灵网状态就能跑
- 剧情关系靠运行时自然生长

这样不会把开放世界绑死。

## 当前推荐工作流

1. 先在前端灵弦池补完整卡
2. 再在角色条目里只声明当前挂载
3. 最后让运行时把挂载摘要注入 AI

一句话：

`卡池在前端，挂载在角色，理解靠注入。`
