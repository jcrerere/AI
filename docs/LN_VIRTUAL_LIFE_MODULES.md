# LN 虚拟生活模块设计

## 目标

LN 不只承接剧情推进，还要承接“玩家在城市里真的能生活”的部分。

这层系统的目标不是把所有内容都塞进正文，而是：

- 对戏推进负责“到达某个场景”
- 场景动作桥负责“亮起可进入的玩法入口”
- 具体玩法页面负责“局部渲染和结算”
- 结算结果再回写到正文、变量和账本

这样既能保留沉浸感，也不会让每一轮对戏都膨胀成大段说明。

## 核心原则

### 1. 世界里没有“数值名词”，系统里可以有隐藏值

例如衣物系统里，世界中的人不会说“这件衣服魅力值 +8”。
他们只会觉得：

- 这套很体面
- 这套很挑逗
- 这套像空姐
- 这套像制服
- 这套适合夜场
- 这套不适合追杀任务

所以实现上要分两层：

- 表层：风格印象、场合建议、视觉标签、身份联想
- 底层：隐藏权重和适配值

隐藏值只给系统判定，不直接在世界观里裸露给玩家。

### 2. 模块只在“进入场景时”渲染

LN 不该一直把所有店铺、地铁、黑赛、餐厅都挂在上下文里。

正确方式是：

- 平时只保存轻量锚点和路由
- 到达地点后，动作桥亮起入口
- 玩家点击后，才生成那一页的具体内容
- 结算结束后，只把结果摘要写回正文和变量

这就是文字版的“远景模糊，近景渲染”。

### 3. 先做“模块原型”，再做“命名场所”

不能要求每家店都先写好。

应该先有：

- 店铺原型
- 商品池
- 地区偏好
- 刷新规则
- 价格规则

命名场所只是覆盖层，不是唯一来源。

也就是说：

- 写了名字的店，可以有专属库存和偏好
- 没写名字的店，也能按地区和类型即时生成

### 4. AI 负责判定入口，前端负责渲染细节

AI 最适合做：

- 现在到的是不是商店
- 现在是不是进入黑赛盘口
- 现在是不是上了地铁站台
- 现在是不是进入服务场所

AI 不适合自由编造：

- 页面结构
- 完整物品清单
- 复杂赔率表
- 地铁线路数据

所以 LN 的最佳结构是：

- AI 输出轻量路由
- 前端按路由查模块注册表
- 模块自己生成具体内容

## 模块总表

### A. 购物模块

这是最大的基础模块，后续绝大多数生活场景都会落到这里。

先拆成这些子类：

- 服装店
- 芯片店
- 灵械/义体诊台
- 毒品交易点
- 烟草与精神品零售点
- 奢侈品与礼物店
- 日用品店

每类共用统一购物框架，但商品生成规则不同。

### B. 黑赛下注模块

这是诺丝区高优先玩法。

它不是普通购物页，而是带结果波动的风险系统。

需要有：

- 盘口
- 参赛者
- 倍率
- 风险说明
- 结算结果
- 历史输赢记录

后续可加：

- 买情报改赔率
- 盘口封盘
- 热门选手爆冷
- 串关

### C. 地铁与城轨模块

这个模块的目标不是“交通工具本身”，而是城市地图感。

要表达的是：

- 城市是连通的
- 区域之间有不同线路
- 玩家是在真实移动，而不是瞬移

后续可以接：

- 区域到站广播
- 夜间停运
- 特殊区限制进站
- 票价差异

### D. 服务场所模块

这一类承接红灯区、陪伴服务、会所等玩法。

它和普通购物不一样，因为结算结果不只是买到物品，而是：

- 关系变化
- 身份暴露风险
- 心情变化
- 依赖与成瘾
- 社交线索
- 特殊剧情挂钩

先做成独立模块，不混进普通商店。

### E. 餐厅与约会模块

这个模块的价值不是回血，而是补“虚拟生活感”。

它适合承接：

- 和 NPC 出门
- 地区消费层级差异
- 对话氛围变化
- 关系推进
- 生活感补足

### F. 诊疗与改写模块

这是“老维式民间大神”最适合落的模块。

要承接：

- 灵械维护
- 芯片调校
- 非法改写
- 临时解限
- 风险提示

它和普通芯片商店不同：

- 商店是买
- 诊台是装、改、拆、修

## 购物模块详细规则

### 1. 店铺不等于固定清单

每个购物入口由四部分决定：

- 地区
- 店铺类型
- 店铺层级
- 随机种子

例如：

- 诺丝区 + 服装店 + 中高端 + 当前刷新周期
- 艾瑞拉区 + 制服店 + 官方管制 + 当前刷新周期

这样能生成稳定但会更新的货架。

### 2. 店铺层级

建议统一做 4 档：

- `street`
  - 便宜、杂、命中感强、质量不稳定
- `standard`
  - 常规商业店
- `premium`
  - 高端、审美稳定、价格高
- `elite`
  - 稀有、身份导向、带门槛

层级影响：

- 价格
- 稀有度
- 款式权重
- 商品更新速度
- 是否出现定制品

### 3. 商品刷新

商品不应永久固定。

建议做：

- 常规刷新：按天或按周
- 区域刷新：换区后池子不同
- 事件刷新：节庆、赛季、区域活动、剧情节点

实现上不要保存整家店的完整库存，只保存：

- 店铺种子
- 当前刷新期
- 已购买记录

然后按算法即时生成。

### 4. 服装不直接叫“魅力值”

服装内部建议拆成隐藏属性：

- `presentation`
  - 外在表现力
- `provocation`
  - 挑逗感
- `authority`
  - 权威/制服感
- `elegance`
  - 正式与礼仪感
- `mobility`
  - 行动便利度
- `comfort`
  - 日常舒适度
- `exposure`
  - 暴露度
- `contextBias`
  - 适合场景

表层只显示：

- 风格标签
- 适用场合
- 穿着印象
- 周边人的可能反应

例如：

- `常服`
  - 舒适、日常、不会引人注意
- `空姐制服`
  - 制服感强、身份感强、挑逗度中高、任务场合受限
- `情趣内衣`
  - 暴露度高、挑逗度高、场景适配非常窄
- `JK`
  - 年轻化、学院感、社交吸睛、正式性低
- `礼服`
  - 仪式感高、正式感高、行动便利度低

### 5. 服装逻辑不能只看“最高值”

这是最重要的一条。

NPC 或系统选衣服时，不该简单选“presentation 最高”的。

而应该按场景模板算适配：

- 追杀任务
  - 更看 `mobility / concealment / authority`
- 正式宴会
  - 更看 `elegance / presentation`
- 夜场引诱
  - 更看 `provocation / exposure / roleplayTags`
- 日常逛街
  - 更看 `comfort / presentation`

这样礼服就不会莫名出现在追杀任务里。

## 黑赛下注模块详细规则

### 1. 黑赛不是纯随机转盘

黑赛应由这些因素共同决定：

- 参赛者基础强度
- 当期状态
- 场地偏向
- 热门度
- 盘口修正
- 随机扰动

这样结果才会“有波动，但不是胡来”。

### 2. 赔率不只是装饰

赔率要反映：

- 强弱差
- 市场热度
- 爆冷风险
- 玩家是否买到内幕信息

所以赔率可做动态：

- 初盘
- 临盘
- 封盘

### 3. 下注结算要进统一账本

结果必须回写：

- 钱包
- 财务流水
- 黑赛历史
- 当前场景摘要

这样它就是 LN 经济系统的一部分，不是独立小游戏。

## 芯片 / 毒品 / 烟草 / 灵械模块差异

这些不能共用一套“买了就结束”的规则。

### 芯片

重点在：

- 兼容性
- 安装槽位
- 风险
- 是否合法
- 是否需要诊台安装

### 毒品

重点在：

- 即时效果
- 副作用
- 成瘾性
- 区域合法性
- 执法风险

### 烟草和精神品

比毒品轻，但要有：

- 短时状态影响
- 社交场景加成
- 依赖风险

### 灵械/义体

重点不在“购买”，而在：

- 调试
- 维修
- 换件
- 解限
- 非法改写

## 服务场所模块规则

红灯区相关玩法不应只是“花钱 -> 文本结束”。

建议最少有这些维度：

- 服务类型
- 身份层级
- 风险等级
- 对应风格
- 关系后果
- 法域后果

返回结果不只是一段文字，还可以是：

- 获得线索
- 触发人物线
- 增加依赖
- 产生欠账
- 上升热度

## 餐厅与约会模块规则

这一类重点不是加属性，而是增强生活感。

可以只做轻量结算：

- 消费档次
- 氛围
- NPC 偏好
- 关系变化
- 触发的对话主题

这样它占用很少 token，但能明显提高城市沉浸感。

## Token 控制方案

这是整套系统最关键的工程原则。

### 1. 正文不直接带完整模块数据

正文只保留：

- 当前地点
- 当前动作接口
- 极短摘要

例如：

- 你走进了诺丝区的一家服装店，货架灯正打在悬挂的制服和礼服上。

不要在正文里塞：

- 20 个商品名
- 价格表
- 稀有度
- 线路总图

### 2. 具体列表由前端即时生成

玩家点开以后，前端按当前地点和模块类型生成：

- 服装列表
- 黑赛盘口
- 地铁线路
- 服务菜单

### 3. 世界书只保存锚点和规则，不保存全量实例

世界书里更适合保存：

- 地区偏好
- 店铺原型
- 特殊命名场所
- 合法性和法域规则

不适合保存：

- 每家店永久库存
- 每次盘口详情
- 每趟列车完整状态

## 实现顺序

### 第一阶段

- 完善动作桥注册表
- 将 `shop / black_race_bet / metro_route` 从“单独实现”升级为“通用模块路由”

### 第二阶段

- 先做购物原型层
- 补服装店的隐藏属性和刷新规则
- 同时保留通用店铺兜底生成

### 第三阶段

- 把黑赛从当前版本升级为“动态盘口”
- 接赔率、状态、热度和历史结算

### 第四阶段

- 做诊台模块
- 让芯片 / 灵械 / 非法改写走另一套规则

### 第五阶段

- 做服务场所模块
- 再做餐厅与约会模块

## 当前最推荐先做的 3 个

- `服装店`
  - 最适合验证“隐藏值 + 表层标签 + 刷新货架”
- `黑赛下注`
  - 最适合验证“特殊规则模块”
- `地铁线路`
  - 最适合验证“地图感和区域穿透”

如果这 3 个做稳，后面的毒品、烟草、服务场所、餐厅、诊台都能沿这套架构接进去。
